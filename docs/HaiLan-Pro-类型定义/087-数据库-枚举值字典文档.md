#### 1.2 项目愿景
打造极致隐私、智能陪伴、品质合规、全场景覆盖的情趣健康生活管理平台，为用户提供安全、专业、高端的健康生活体验。

#### 1.3 核心价值主张
- **极致隐私**：双重加密、隐私浏览模式及伪装发货机制

### 4. 枚举值使用规范

#### 4.1 数据库DDL中使用

```sql
-- PostgreSQL枚举类型定义（可选，也可使用VARCHAR+CHECK约束）
CREATE TYPE user_status_enum AS ENUM (
    'active',
    'inactive',
    'banned',
    'deleted'
);

CREATE TYPE order_status_enum AS ENUM (
    'pending',
    'paid',
    'shipped',
    'completed',
    'cancelled'
);

-- 使用枚举类型
CREATE TABLE users (
    status user_status_enum DEFAULT 'active' NOT NULL
);
```

#### 4.2 TypeScript代码中使用

```typescript
/**
 * 将数据库枚举值转换为TypeScript枚举
 */
export const UserStatusMap: Record<string, UserStatus> = {
  active: UserStatus.ACTIVE,
  inactive: UserStatus.INACTIVE,
  banned: UserStatus.BANNED,
  deleted: UserStatus.DELETED
};

/**
 * 枚举显示名称映射
 */
export const UserStatusLabel: Record<UserStatus, string> = {
  [UserStatus.ACTIVE]: '活跃',
  [UserStatus.INACTIVE]: '未激活',
  [UserStatus.BANNED]: '已封禁',
  [UserStatus.DELETED]: '已删除'
};

/**
 * 获取枚举显示名称
 */
export function getEnumLabel(enumObj: any, value: string): string {
  const key = Object.keys(enumObj).find(k => enumObj[k] === value);
  return key ? enumObj[`${key}_LABEL`] || value : value;
}
```

#### 4.3 Vue组件中使用

```vue
<template>
```

### 5. 枚举值扩展规范

#### 5.1 新增枚举值步骤

1. **评估影响范围**：确认新增枚举值对现有数据的影响
2. **数据库迁移**：编写ALTER TABLE语句添加新值
3. **代码同步**：更新TypeScript枚举定义
4. **UI更新**：更新相关下拉框、筛选器
5. **文档更新**：同步更新此文档

#### 5.2 枚举值弃用规范

1. **标记为deprecated**：在注释中标注`@deprecated`
2. **保留兼容性**：不在数据库中删除，仅代码中禁用
3. **迁移路径**：提供替代枚举值和迁移建议
4. **通知开发**：在团队中同步弃用信息

```typescript
/**
 * @deprecated 使用 ORDER_REFUNDING 替代
 * 订单退款状态
 */
export const ORDER_REFUND = 'order_refund';
```
