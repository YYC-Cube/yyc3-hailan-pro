## 2. Figma命名约定

## 3. 组件库和样式库配置

### 3.1 共享样式创建步骤(颜色/文字/效果)

**颜色样式**:
1. 选择画布颜色 → 右侧面板"资源" → 颜色 → "创建样式"
2. 按命名规则命名
3. 应用到所有元素

**文字样式**:
1. 选择文本框 → 设置字体/字号/行高
2. 资源面板 → 文字 → "创建样式"
3. 命名 → 应用,支持批量样式刷

**效果样式**:
1. 给元素添加阴影/模糊
2. 资源面板 → 效果 → "创建样式"
3. 命名 → 保存,跨文件复用需要"发布样式库"

**样式库共享**:
1. 文件 → "发布资源"
2. 选择要共享的样式
3. 设置版本 → 发布
4. 团队成员通过"资源 → 库 → 导入"使用

### 3.2 组件库配置步骤

**原子组件创建**:
1. 选择设计元素 → 右键"创建组件"
2. 按命名规则命名
3. 拖入"组件库"框架

**组件变体设置**:
1. 选择组件 → 右侧面板"属性" → "创建变体"
2. 添加属性(例如,"状态"、"尺寸")
3. 配置每个变体样式 → 绑定属性

**组件属性绑定**:
1. 选择组件内元素(例如,按钮文字)
2. 右侧面板"属性" → "绑定属性"
3. 关联变体属性(例如,状态对应文字颜色)

**组件库发布**:
1. 文件 → "发布资源"
2. 选择组件 → 设置版本 → 发布
3. 团队可实时同步组件更新
4. 支持"覆盖/保留"本地修改

**分子组件组合**:
1. 使用原子组件(按钮/输入框)组合成分子组件
2. 创建组件 → 保留原子组件变体关联
3. 确保联动修改

### 3.3 样式/组件更新规则

**小修改(细节优化)**:
- 更新样式/组件 → 重新发布
- 版本号"次版本+1"(例如,v1.0→v1.1)

**大修改(结构调整)**:
- 复制原组件/样式 → 修改后重命名
- 发布新主版本(例如,v1.0→v2.0)
- 保留旧版本兼容性

## 4. 深色模式适配规则

### 4.1 核心颜色适配

| 颜色系统       | 浅色模式主色       | 深色模式主色       | 适配原则                  | 对比度要求       |
|------------|--------------------|--------------------|---------------------------|------------------|
| 主色-深蓝 | #1A365D(900) | #2A4B7D(800) | 降低饱和度、提高亮度      | 文字到背景≥4.5:1 |
| 辅助-极光紫 | #6B46C1(700) | #7B56D1(600) | 降低饱和度、避免刺眼      | 交互元素≥3:1     |
| 语义-成功绿 | #48BB78(600) | #58CB88(500) | 提高亮度、保持可识别性      | 同浅色模式       |
| 中性-背景  | #FFFFFF(0)/#F7FAFC(50) | #121212(950)/#1A1A1A(900) | 背景使用深灰系统、避免纯黑    | 文字到背景≥4.5:1 |
| 中性-文字  | #1A1A1A(900)/#4A5568(700) | #F7FAFC(50)/#A0AEC0(300) | 主文字浅灰、次要文字更浅  | 同浅色模式       |

### 4.2 组件/元素适配规则

**卡片/容器**:
- 浅色模式:白色 + 阴影
- 深色模式:#1A1A1A + 低透明度阴影(0 4px 6px rgba(255,255,255,0.05))

**输入框/按钮**:
- 浅色模式:边框#E0E0E0
- 深色模式:边框#3A3A3A
- 背景色同步深色梯度
- 文字颜色反转

**隐私遮罩**:
- 浅色模式:模糊 + 白色遮罩
- 深色模式:模糊 + 深灰遮罩(rgba(12,12,12,0.6))
- 保持遮罩可识别性

**图标/分割线**:
- 浅色模式:#4A5568
- 深色模式:#A0AEC0
- 透明度同浅色模式(例如,60%)

### 4.3 Figma深色模式配置步骤

1. 点击文件 → "设置" → "深色模式" → "启用"
2. 为每个颜色样式创建"深色模式变体"
3. 选择对应深色梯度 → 保存
4. 为组件添加"外观"属性(浅色/深色)
5. 配置每个外观下的样式
6. 绑定颜色样式的深色模式变体
7. 在画布右上角切换"浅色/深色模式"
8. 所有元素自动适配,无需手动修改

## 5. 跨平台交互标准

### 5.1 移动端手势 ↔ PC鼠标操作对应

| 交互行为       | 移动端操作                | PC操作                  | 统一反馈效果              |
|----------------|---------------------------|---------------------------|---------------------------|
| 元素选择       | 单次点击                      | 左键点击                  | 元素高亮/轻微缩放(0.98x) |
| 元素确认       | 双击/长按200ms            | 左键双击/右键确认         | 确认动画(勾选/淡入)|
| 内容删除       | 左滑元素                  | 右键元素 → 删除/左键删除按钮 | 删除动画(淡出/左滑消失)|
| 内容收藏       | 右滑元素/点击收藏图标     | 左键点击收藏图标          | 收藏图标颜色变化(极光紫) + 震动反馈 |
| 视图切换       | 左右滑动页面              | 鼠标滚动/左右方向键       | 页面过渡动画(300ms滑动) |
| 内容缩放       | 捏合/张开             | 鼠标滚动/按住Ctrl+滚动    | 平滑缩放,带最大/最小限制 |
| 下拉刷新       | 下拉页面80px              | 鼠标向下滚动/点击刷新按钮 | 刷新旋转动画,加载完成回弹 |
| 长按操作       | 长按元素调用操作菜单      | 右键元素调用操作菜单      | 菜单淡入,带阴影,点击空白关闭 |

### 5.2 跨平台动画时长/反馈标准

**动画时长**:
- 所有平台页面过渡:200-300ms
- 微交互(按钮/图标):150-200ms
- 加载动画:无限循环(匀速)

**交互反馈**:
- 按钮点击:所有"缩放0.95x → 恢复"
- 输入框焦点:所有"边框颜色变化(深蓝) + 轻微发光"
- 错误状态:所有"轻微震动(3次) + 边框变红"

**加载状态**:
- 所有平台加载使用统一旋转器(品牌元素变形)
- 骨架屏占位符样式统一
- 避免不同平台加载样式混淆

**空状态/错误状态**:
- 跨平台空状态插画/文案统一
- 错误状态提示文案 + 解决方案建议统一
- 所有带"重试"按钮

## 6. 设计评审定量评分

### 6.1 评分标准(8个关键点·总分100分)

**评审结果等级**:
- 优秀(≥90)
- 良好(80-89)
- 合格(70-79)
- 不合格(<70)

| 评审点               | 分数 | 评分标准(量化) | 扣分规则                  |

## 7. 开发交接问答

### 7.1 Q1:如何让Figma颜色/间距/圆角样式直接对齐Tailwind CSS?

**答案**:
1. 设计阶段,按Tailwind规则命名颜色/间距/圆角(例如,颜色#1A365D命名为`primary-900`,间距24px命名为`lg`)
2. 在Figma中导出"CSS变量表"(包含所有设计令牌)
3. 前端直接配置变量到Tailwind.config.js
4. 组件样式直接使用Tailwind类名(例如,`bg-primary-900 rounded-lg p-4`)
5. 无需重新定义样式,实现设计开发1:1对齐

### 7.2 Q2:如何将Figma组件多个状态(默认/悬停/按下)传递给开发?

**答案**:
1. 为每个组件创建"状态描述表"(包含状态名称、样式、触发条件)
2. 附加到Figma交接文档
3. 在Figma中为组件添加"交互描述"(右键组件 → 添加注释)
4. 标注每个状态的CSS样式(例如,悬停`bg-primary-800 cursor-pointer`)
5. 前端基于Vue/React组件状态实现(例如,`:class="{ 'bg-primary-800': isHover }"`)
6. 设计提供每个状态的Figma标注值(颜色/尺寸/阴影)

### 7.3 Q3:如何让Figma标注简洁高效,避免重复开发询问?

**答案**:
1. 使用Figma"标注插件"(例如,Measure/Figma to Code)
2. 启用"自动标注",标注内容包括"尺寸/间距/颜色/字体/行高"
3. 仅标注"关键元素",重复组件(例如,按钮)仅标注一次
4. 附加组件库链接
5. 标注跨平台页面的"断点尺寸"
6. 标注时区分"移动端/平板/桌面",避免尺寸混淆
7. 隐藏设计辅助图层(例如,网格/参考线)
8. 仅保留开发实际需要的元素,防止标注干扰

### 7.4 Q4:如何将Figma交互动画传递给开发,确保还原准确?

**答案**:
1. 为每个动画创建"动画描述表"(包含动画类型、时长、触发条件、效果)
2. 附加到交接文档
3. 在Figma中录制动画GIF(使用Figma Pro"录制原型"功能)
4. 导出给开发作为参考
5. 简单动画直接标注CSS动画属性(例如,`transition: all 0.3s ease`)
6. 复杂动画(例如,AR/3D)提供动画参考视频
7. 前端使用CSS/JS/Three.js实现
8. 与开发约定"动画通用规则"(例如,所有过渡都是`ease-in-out`,时长300ms)
9. 避免动画还原混淆

### 7.5 Q5:如何导出Figma图标/图片资源满足前端开发需求?

**答案**:
1. 图标全部导出为**SVG格式**(在Figma中选择图标 → 导出 → 选择SVG → 勾选"作为组件导出")
2. 按类别组织(例如,icon-privacy/icon-nav)
3. 前端直接导入使用,支持缩放不失真
4. 图片全部导出为**PNG/WebP格式**(分辨率2x,适配视网膜屏幕)
5. 导出前压缩
6. 附加图片命名规则(例如,img-home-hero-1440px)
7. 在Figma中创建"资源导出文件夹"
8. 将所有需要导出的资源放入其中
9. 统一导出,避免资源遗漏

### 7.6 Q6:设计迭代后,如何快速同步给开发,避免版本混淆?

**答案**:
1. 设计迭代后,在Figma中"发布新资源版本"
2. 标注更新内容(例如,"修改按钮悬停颜色","优化购物车布局")
3. 向开发发送"版本更新通知"
4. 附加更新点描述 + Figma文件链接
5. 在Figma中使用"标注"标记更新元素(红色标注)
6. 开发直接查看标注位置,快速定位修改点
7. 保留设计历史版本
8. 如果开发需要回滚,可快速导出旧版本资源

### 7.7 Q7:如何让开发准确还原移动端手势交互(例如,滑动删除/下拉刷新)?

**答案**:
1. 在Figma原型中实现完整手势交互
2. 开发可直接操作原型体验
3. 为每个手势交互创建"交互流程图"
4. 标注触发条件(例如,滑动距离≥50px触发删除)
5. 标注反馈效果、动画时长
6. 前端基于移动端原生事件实现(例如,`touchstart/touchmove/touchend`)
7. 设计提供手势触发阈值(例如,下拉80px触发刷新)
8. 设计提供动画参数
