## 一、项目概述与规划设计

### 1.1 项目名称与定位

**项目名称：** HaiLan Pro - 智能情趣生活解决方案

**项目定位：** 高端、智能、隐私至上的成人健康生活平台，结合AI技术与现代设计美学，提供专业、合规、个性化的情趣健康产品与服务。

**项目愿景：** 打造"海蓝(HaiLan)"——新一代高端、私密、智能的情趣健康生活管理平台。项目不局限于传统的电商交易，而是通过 PWA (Progressive Web App) 技术，结合 AI 智能辅助与物联网，为用户提供从生理健康到心理愉悦的全方位解决方案。

### 1.2 核心价值主张

* **极致隐私：** 采用双重加密、隐私浏览模式及伪装发货机制，确保用户数据与购买行为的绝对安全。
* **智能陪伴：** 内置基于 LLM 的 AI 情感与生理健康顾问，提供非评判性的专业咨询。
* **品质合规：** 严格筛选符合医疗级标准的商品，界面设计去低俗化，采用高端"海蓝蓝"视觉调性，强调健康与美学。
* **全场景覆盖：** PWA 端支持离线浏览、桌面安装及无缝推送，可平滑接入微信/支付宝小程序生态。

### 1.3 核心特色

- **智能推荐系统：** 基于AI算法的个性化商品推荐
- **隐私极致保护：** 全链路隐私保护方案
- **健康导向：** 强调产品的情感连接与健康价值
- **体验创新：** AR/VR产品预览、智能配对等创新功能
- **教育内容：** 专业的性健康教育内容社区

### 1.4 目标用户画像

* **核心群体：** 25-40岁，中高收入，追求生活品质，注重隐私的都市人群（夫妻、情侣、单身人士）。
* **潜在群体：** 关注性健康、有生理康复需求（如产后修复）的用户。

## 二、技术架构设计

### 2.1 前端技术栈

* **核心框架：** Uni-app (Vue 3.5+ TypeScript) —— 实现一套代码多端发布（H5/PWA + 小程序）。
* **构建工具：** Vite 6.x —— 提供极速的开发体验和热更新。
* **状态管理：** Pinia —— 轻量级、类型安全的状态管理。
* **UI 组件库：** Tailwind CSS + Uni-ui —— 快速构建响应式、原子化的 UI。
* **PWA 核心：** Workbox —— 处理 Service Worker 注册、缓存策略和离线能力。
* **数据可视化：** ECharts (用于个人健康报告展示)。

### 2.2 后端技术栈

* **运行环境：** Node.js 20 LTS。
* **核心框架：** NestJS 10.x —— 模块化架构，支持依赖注入，便于微服务扩展。
* **ORM：** Prisma —— 类型安全的数据库访问，支持 PostgreSQL/MySQL。
* **缓存与消息：** Redis (缓存/Session) + RabbitMQ (订单异步处理/解耦)。
* **AI 集成：** LangChain + OpenAI API / 国产大模型 API (用于智能客服与推荐)。

### 2.3 基础设施

* **数据库：** PostgreSQL 16 (主业务库) + MongoDB 17 (日志与用户行为分析)。
* **文件存储：** AWS S3 / 阿里云 OSS (图片/视频资源)。
* **搜索服务：** Elasticsearch (商品全文检索)。
* **部署：** Docker + Kubernetes (K8s) + Nginx (反向代理与 SSL 终止)。

### 2.4 系统模块架构

```
1. 用户系统（多重隐私保护）
2. 商品管理系统（智能分类与标签）
3. 智能推荐引擎
4. AR/VR体验模块
5. 隐私购物系统
6. AI健康助手
7. 内容社区（UGC/PGC）
8. 会员与积分体系
9. 多渠道客服系统
10. 数据分析后台
```

## 三、核心功能设计

### 3.1 用户隐私保护系统（重点）

#### 3.1.1 隐私层级设置

- **访客模式：** 无需注册，浏览基础商品
- **轻量账户：** 仅需邮箱，无个人身份信息
- **隐私账户：** 端到端加密通信，数据本地存储选项
- **会员账户：** 完整功能，强化隐私保护

#### 3.1.2 隐私保护功能

- **隐身模式：** 不记录浏览历史
- **包裹伪装：** 中性化包装，无标识物流单
- **支付隐私：** 支持虚拟信用卡、数字货币
- **数据控制：** 用户可随时导出/删除全部数据
- **本地存储：** 敏感数据可选择仅存本地设备
- **应用伪装/隐私模式：** 一键切换计算器、笔记或新闻界面
- **无痕浏览：** 本地不保存浏览历史，应用图标可自定义
- **生物识别锁：** 支持指纹/FaceID 验证进入 App

### 3.2 智能商品系统

采用"场景导向 + 科学分类"体系，避免露骨词汇：

## 四、数据库与类型定义

### 4.1 核心数据表设计

* **users:** 用户基础信息、加密后的支付密码、隐私设置。
* **profiles:** 用户画像（年龄段、偏好标签、健康状态）。
* **products:** 商品 SPU 信息、媒体资源、AI 描述标签。
* **product_skus:** 商品 SKU 信息（价格、库存、规格）。
* **orders:** 订单主表（含虚拟收货人/脱敏地址字段）。
* **order_items:** 订单子表。
* **iot_devices:** 绑定的智能设备信息、固件版本。
* **health_records:** 用户健康打卡/设备使用记录（加密存储）。
* **knowledge_articles:** 科普文章内容。

### 4.2 分类定义

采用"场景导向 + 科学分类"体系，避免露骨词汇：

* **Level 1:**
  * `CARE` (健康护理 / Care Products)
  * `PLAY` (愉悦探索 / Pleasure Toys)
  * `INTIMACY` (亲密增进 / Intimacy Enhancers)
  * `LINGERIE` (贴身服饰 / Lingerie)
  * `SMART` (智能硬件 / Smart Tech)
* **Level 2 (示例 - under PLAY):**
  * `EXTERNAL` (外用探索)
  * `INTERNAL` (内用体验)
  * `COUPLES` (伴侣互动)

## 五、UI设计系统与交互设计

### 5.1 设计系统

* **主色调：** 海蓝蓝 (#0056b3) 与 深夜灰 (#1a1a1a) 结合，营造沉稳、高端、科技感。
* **辅助色：** 极光紫 (用于强调互动)、医疗白 (背景)。
* **圆角风格：** 大圆角 (24px+)，柔和友好，无攻击性。
* **字体：** 无衬线字体，强调清晰易读。

### 5.2 设计原则

- **优雅简约：** 采用高端电商设计语言
- **中性色调：** 主色调：深蓝、浅灰、白色
- **克制表达：** 避免露骨图片，使用艺术化渲染
- **明确指引：** 清晰的导航与分类

### 5.3 核心页面列表

1. **启动页：**
    * 支持显示为"系统工具"或"海蓝健康" Logo。
    * 生物识别解锁弹窗。

2. **首页：**
    * **顶部：** 隐蔽的搜索栏（默认显示为"搜索..."，点击后联想词加密显示）。
    * **金刚区：** 图标化分类入口（如：独处时光、双人互动、健康调理）。
    * **AI 每日签：** 基于日历的健康/情感建议。
    * **瀑布流：** 商品推荐流（图片默认模糊，长按/点击清晰，防偷窥）。

3. **商品详情页：**
    * **沉浸式轮播：** 支持视频演示。
    * **隐私购买选项：** 默认勾选"隐私发货"、"面单脱敏"。
    * **AI 解读：** 商品参数通俗化解读（如：分贝值、材质安全度）。

4. **购物车/结算页：**
    * **一键加密模式：** 开启后商品名称变为"生活日用品"。
    * **支付：** 集成微信支付/支付宝，支持生成虚拟订单号。

5. **AI 伴侣页：**
    * 对话式界面，支持语音输入。
    * 提供"性福测试"、"体质评估"等互动小游戏。

6. **个人中心：**
    * **数据看板：** 健康指数、快乐指数（基于智能硬件数据）。
    * **隐私管家：** 浏览记录清理、账号安全中心、伪装设置。
    * **订单服务：** 仅显示脱敏后的订单状态。

7. **无人售货机地图页 (O2O)：**
    * 基于地理位置展示附近 24h 智能柜。
    * 支持扫码开门、线上下单柜机取货。

### 5.4 特殊交互优化

* **防误触与防窥屏：** App 切换至后台后，再次进入需验证；在设置中可开启"摇一摇模糊屏幕"。
* **智能客服触发：** 在商品详情页停留超过 10 秒或多次添加购物车又移除，自动弹出 AI 小助手询问"是否需要选型建议"。
* **PWA 安装引导：** 根据浏览器行为，智能弹出"添加到主屏幕"提示，提供类原生 App 体验。

### 5.5 响应式设计

- **移动优先：** PWA支持离线浏览
- **自适应布局：** 从手机到桌面完美适配
- **手势优化：** 移动端手势操作
- **性能优化：** 图片懒加载，关键资源预加载

## 十三、开发执行与闭环

### 13.1 开发方法论与策略

#### 13.1.1 敏捷开发 + Privacy by Design (隐私设计)

采用 **Scrum 敏捷开发模式**，以 2 周为一个 Sprint。在 User Story (用户故事) 的定义中强制加入"隐私验收标准"。

* **开发原则：** "最小权限原则"。后端接口仅返回前端必需的数据，禁止返回完整敏感信息。
* **分支管理：** Git Flow 工作流。
  * `master/main`: 生产环境代码，受保护。
  * `develop`: 开发主分支。
  * `feature/*`: 功能分支。
  * `hotfix/*`: 紧急修复分支（如安全漏洞）。

#### 13.1.2 代码规范与协作

* **前端：** ESLint + Prettier 强制统一风格，TypeScript 严格模式 (`strict: true`)，确保类型安全防止隐私数据类型错误。
* **后端：** Husky 提交前检查，API 接口遵循 RESTful 规范，所有涉及隐私的接口必须通过 Swagger/Yaml 标注 `security: [{ BearerAuth: [] }]`。

### 13.2 测试与质量保证 (QA)

鉴于产品的敏感性，QA 环节需增加"安全与隐私专项测试"。

#### 13.2.1 功能测试

* **电商闭环：** 注册 -> 加密浏览 -> 加密下单 -> 支付回调 -> 订单状态流转 -> 评价。
* **多端适配：** 重点测试小程序端与 PWA 端的交互一致性（特别是支付授权差异）。

#### 13.2.2 安全与隐私测试

* **数据泄露检测：**
  * 拦截 API 请求，检查响应体是否包含明文的用户手机号、地址。
  * 检查本地存储，确认无明文 Token 或敏感记录。
* **伪装模式测试：**
  * 测试"一键伪装"响应速度 < 200ms。
  * 测试应用切换后台后，再唤起时的锁屏逻辑。
* **渗透测试：** 使用 OWASP ZAP 对 API 进行扫描，防止 SQL 注入、XSS 攻击。

#### 13.2.3 性能测试

* **Lighthouse 跑分：** 确保 PWA 首屏加载 (LCP) < 1.5s。
* **高并发模拟：** 模拟秒杀场景，测试 Redis 缓存击穿保护及消息队列削峰填谷能力。

### 13.3 部署与运维

#### 13.3.1 CI/CD 流水线 (基于 GitLab CI 或 GitHub Actions)

1. **代码提交：** 触发 Pipeline。
2. **代码检查：** Lint + Unit Test (Jest)。
3. **构建镜像：** Docker Build (Backend & Frontend)。
4. **部署预发布：** 自动部署到 Staging 环境，运行 E2E 测试。
5. **人工审批：** 产品经理与安全负责人审批。
6. **部署生产：**
    * 后端：K8s 滚动更新，保持服务不中断。
    * 前端：CDN 刷新预热。
    * 数据库：执行 Prisma Migrate 迁移脚本。

#### 13.3.2 基础设施安全

* **WAF (Web应用防火墙)：** 开启防 SQL 注入、防 XSS 规则。
* **DDoS 防护：** 接入云厂商的高防 IP。
* **数据库备份：** 每日全量备份 + Binlog 实时备份，备份文件同样进行加密存储。

### 13.4 迭代与数据闭环 (Intelligent Loop)

作为"智能应用"，开发上线只是开始，数据驱动的迭代是核心。

#### 13.4.1 埋点与数据分析

* **合规埋点：** 采集用户行为路径（如：某品类详情页跳出率高），但严禁关联用户身份信息 (PII)。
* **转化漏斗分析：** 监控"浏览 -> 加购 -> 下单 -> 支付"各环节流失率，优化 UI 交互。

#### 13.4.2 AI 模型持续训练

* **反馈收集：** 用户在 AI 问答页面的点赞/点踩数据。
* **模型微调：** 定期将脱敏后的精选问答数据导入向量数据库，优化 AI 回答的准确性与合规度。

#### 13.4.3 A/B 测试机制

* **页面级 A/B：** 对比"传统列表页"与"场景化推荐页"的转化率。
* **文案级 A/B：** 测试不同风格的商品文案（如"医学描述" vs "感性描述"）对点击率的影响。

### 13.5 项目风险控制与合规闭环

在整个开发闭环中，必须植入合规检查点：

1. **内容合规：** 所有上架商品图片、描述、视频需先经 AI 审核系统（腾讯云/阿里云内容安全）+ 人工复审双重机制，确保不触犯低俗红线。
2. **支付合规：** 确保支付通道支持情趣品类，防止资金冻结（通常需要对接特定聚合支付或使用具备此资质的渠道）。
3. **数据合规：** 严格遵守《个人信息保护法》及 GDPR（如出海），提供明确的"注销账号"功能，并物理删除用户数据。
